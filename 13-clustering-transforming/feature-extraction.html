13.6. Alternative Bases for Feature Extraction#

In this section, we will use an alternative basis to create a classification function from a data set with two classes. We will take advantage of one of the data sets available in the scikit-learn library. Scikit-learn has a module called datasets that contains several well known data sets that are useful for statistics and machine learning. For a description of these data sets, see https://scikit-learn.org/stable/datasets/toy_dataset.html.

In this section, we will use one of the oldest and most famous data sets for classification problems, the Iris data set. This is a relatively simple data set, and we are going to simplify it more for the purposes of this section. In Chapter 15, we will formalize and generalize the techniques developed in this section and apply them to a larger data set associated with more contemporary issues.

According to the DESCR field of the scikit-learn iris dataset object:

This is perhaps the best known database to be found in the
pattern recognition literature.  Fisher's paper is a classic
in the field and is referenced frequently to this day....  The
data set contains 3 classes of 50 instances each, where each
class refers to a type of iris plant.

(To see the full description, run print(iris['DESCR'] after loading the Iris data set as shown below.)

The data set is from Robert Fisher’s paper “The use of multiple measurements in taxonomic problems,” Annual Eugenics, 7, Part II, 179-188 (1936).

We can load it as follows:

It would be nice if we could use a single feature to perform classification. One possible approach is to use just sepal length or just sepal width. Such an approach is a type of feature selection:

DEFINITION

feature selection#

In feature selection, only a subset of the features present in the data is used.

Feature selection may be used for many different reasons including:

  • simplifying a statistical model,

  • reducing training time for a machine-learning model,

  • reducing the time to collect a sufficiently large body of training data, or

  • removing data that is irrelevant to the problem.

However, feature selection alone is not enough to separate these data classes. For instance, the figure below shows just the sepal length for the Setosa and Versicolor irises. I have jittered the y-value around 1.0 so that it is easier to distinguish the various data points:

However, we can see from the figure that uses both features that these classes can be separated by a line. If we represent these classes using a rotated set of axes, then we can perform data reduction to one-dimension. The resulting features after rotation are linear combinations of the original features. When we use only a single one of these new features, this is an example of feature extraction:

DEFINITION

feature extraction#

In feature extraction, a reduced set of features is created, where the new features are functions of the original features.

In Section 13.3, we saw that the vectors for axes rotated counterclockwise by \(\theta\) degrees are:

\[\begin{align*} \mathbf{e}_{x,\theta} &= \left[~~~\cos \theta ~~ \sin \theta ~~\right], \mbox{ and} \\ \mathbf{e}_{y,\theta} &= \left[ \,-\sin \theta~~ \cos \theta ~~\right]. \end{align*}\]

The function makerot() inputs a value of \(\theta\) in degrees and returns the new basis vectors:

def makerot(theta):
  '''return basis vectors for rotation of theta degrees
  
  Parameters
  ----------
  theta : number
    Rotation in degrees
    
  Returns
  -------
  numpy.ndarray
      Basis vector for first dimension (new x coordinate)
      
  numpy.ndarray
      Basis vector for second dimension (new y coordinate)
  '''
  theta=np.radians(theta)
  return np.array([np.cos(theta), np.sin(theta)]), \
         np.array([-np.sin(theta), np.cos(theta)])

Now let’s make basis vectors with a \(60^\circ\) rotation and visualize the data in the new basis:

print(x60, y60)

On the plot, I also included a line at a value of 2.1 in the new feature. This line can be used for classification of any new data according to the following protocol:

  1. The scalar projection of the new data onto the vector \([ 0.76604444 -0.64278761]^T\) is calculated.

  2. If the scalar projection is less than or equal 2.1, the sample is classified as Setosa. If it is more than 2.1, it is classified as Versicolor.

Here, the value of 2.1 was selected by inspection. The classification regions for the data can also be determined from other techniques, such as from the Voronoi regions in the \(K\)-means algorithm.

If we want to perform classification on all three classes, then using only the first two features is not sufficient, and our approach of choosing rotations via visualization breaks down. In Chapter 15, we present a general approach for feature extraction that can find the features that capture the most significant information about the data, while also reducing dependence across the features. However, more sophisticated tools for working with collections of vectors of data are needed, and these are covered in Chapter 14 in the context of matrices.
