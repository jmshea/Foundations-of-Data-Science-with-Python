12.2. Null Hypothesis Significance Testing for Dependence in Contingency Tables#

When the relative frequencies in a contingency table vary by row or column, that suggests that the categorical data in the rows may depend on the categorical data in the columns, and vice versa. However, the observed dependence may also be caused by randomness in the data sample. To determine whether we can be confident that any observed dependence cannot reasonably attributed to randomness, we can conduct a NHST. The null hypothesis, \(H_0\), will be that the data in the rows and columns are not dependent. We have already defined the summary statistic that we will use, \(C\), so we just need to determine the probability of seeing such a large value of \(C\) under \(H_0\). We can do this using resampling or analysis, each of which we cover below.

12.2.1. Resampling NHST Contingency Test#

Let’s load our data for marijuana use by reported sex of college students in the 2021 BRFSS study and generate the contingency table:

import pandas as pd
import numpy as np
import scipy.stats as stats
import pickle

with open('data/college.pickle', 'rb') as f:
  college = pickle.load(f)

college2=college.dropna()  
contingency = pd.crosstab(college['sex'], college2['marijuana_use'])

expected_contingency = stats.contingency.expected_freq(contingency)
print(expected_contingency)

[[ 8.42553191  2.34042553 11.23404255]
 [ 9.57446809  2.65957447 12.76595745]]

And the chi-squared statistic is:

C = ((contingency.to_numpy() - expected_contingency)**2 \
    / expected_contingency).sum()
print(f'C = {C: .2f}')

C =  2.69

To perform resampling under \(H_0\), we wish to break the dependence between the sex and marijuana_use. As with the NHST for correlation, the dependence will be broken if we randomly assign values of sex to values of marijuana_use. Since we will reuse the exact same data but simply reassign the mapping between the pairs of data, this is a type of permutation test.

We can choose to permute the data for either feature; the result will be the same. Let’s permute the data in the marijuana_use feature. We can call the sample() method with frac=1 to get a sample that is the same size as the original data. The sample() method samples without replacement, so with frac=1, it randomly permutes the data. To carry out the NHST, we recompute the chi-squared statistic for each permuted vector and determine the relative frequency of exceeding the chi-squared statistic of the original data. That relative frequency is an estimate of the \(p\)-value, the probability of observing such a large test statistic under \(H_0\). A Python function to carry out this type of NHST on two features is below:

def nhst_contingency(feature1, feature2,
                     observed_C, num_sims = 1_000):

  ''' 
  Estimate the probability of seeing such a large
  value of the chi-squared statistic under the null 
  hypothesis by using a permutation test
  '''
  
  # Calculate the expected contingency once, outside the loop
  contingency = pd.crosstab(feature1, feature2)
  expected_contingency = stats.contingency.expected_freq(contingency)


  # Set up the counter for how many times we see a chi-squared
  # statistic as large as the original data
  count = 0

  for sim in range(num_sims):
    
    # Calculate the contingency table with one feature permuted
    sample_contingency \
      = pd.crosstab(feature1.to_numpy(),
                    feature2.sample(frac=1).to_numpy())

    sample_C = ((sample_contingency.to_numpy() 
                 - expected_contingency)**2 \
                / expected_contingency).sum()
    
    if sample_C >= observed_C:
      count +=1

  print('Prob. of observing chi-squared value as large as original')
  print(f'data under H0  =~ {count/num_sims: .2g}')

  return count/num_sims

nhst_contingency(college2['sex'],
                 college2['marijuana_use'],
                 C)

Prob. of observing chi-squared value as large as original
data under H0  =~  0.26

0.255

Before interpreting the results, I want to point out one fine point about the simulation that may trip up people building simulations like this for the first time. Note that I have converted each of the Pandas data Series into numpy vectors. The reason that I did this is that if I don’t do it, pd.crosstab() will match up entries with the same index, and the effect of shuffling the data will be negated. By converting to NumPy vectors, data is matched based on position, and the shuffling effect is preserved.

Note that the \(p\)-value is approximately 0.26, which is above our threshold of \(\alpha=0.05\), so we cannot reject the null hypothesis. The data is too small to be able to conclude that the observed dependence between sex and type of marijuana could not be caused by randomness in the sample.

Second example

Let’s try a contingency test with more data. We will compare reported sex with whether the respondent reported exercising in the last 30 days. Let’s jump straight to creating the contingency table:

ex_contingency = pd.crosstab(college['sex'], college['exercised'])

The expected contingency table is

exp_ex_contingency = stats.contingency.expected_freq(ex_contingency)
print(exp_ex_contingency)

 [ 96.28571429 577.71428571]]

The difference table is:

exp_ex_contingency - ex_contingency
exercised no yes

male 16.285714 -16.285714

The raw data seems to suggest that women may be more likely to have exercised in the last 30 days then men, and this time we have 1253 data points. Let’s calculate the chi-squared statistic:

ex_C = ((exp_ex_contingency- ex_contingency.to_numpy() )**2 
        / exp_ex_contingency).sum()
print(f'For sex/exercise table, C = {ex_C:.2f}')

For sex/exercise table, C = 6.95

This is a much larger value of the chi-squared statistic. Let’s determine the \(p\)-value using a permutation test:

nhst_contingency(college['sex'], college['exercised'],
                 ex_C)

Prob. of observing chi-squared value as large as original
data under H0  =~  0.011

0.011

Since the \(p\)-value of 0.011 is less than our significance threshold of \(\alpha=0.05\), the result is statistically significant at the \(p<0.05\) level. We can reject the null hypothesis and conclude that the sex and exercised features are dependent. Note, however, that this test does not allow us to say anything else about how these features depend on each other.

12.2.2. Analytical NHST#

Under \(H_0\), the chi-squared statistic can be shown to be a standard chi-squared random variable. The standard chi-squared random variable requires a single parameter: the number of degrees of freedom (dof). Recall that the number of degrees of freedom for a contingency table with \(r\) rows and \(c\) columns is \((r-1)(c-1)\). Then under \(H_0\), the chi-squared statistic is a chi-squared random variable with \((r-1)(c-1)\) degrees of freedom.

The pdf for a standard chi-squared random variable with different numbers of degrees of freedom is shown below. Note that as the number of degrees of freedom increases, the probability mass moves toward higher values.

import matplotlib.pyplot as plt

x = np.arange(0,10,0.1)
for dof in [1, 2, 4, 6]:
  chi2_rv = stats.chi2(dof)
  plt.plot(x, chi2_rv.pdf(x), label='dof = '+str(dof))
plt.legend();
plt.xlabel('$x$');
plt.ylabel('$f_X(x;$ dof$)$');

Let’s apply this model to determine the \(p\)-value for our first example. Recall that for our example with sex and marijuana_use, we had two rows and three columns, and hence \((2-1)(3-1) = 2 \) degrees of freedom. We can create a chi-squared distribution with 2 dof in SciPy.stats as

# BE CAREFUL stats.chi() is a different type of random variable!
dof = 2
chi2_rv1=stats.chi2(2)

The probability of getting a chi-squared statistic as large as the 2.69 we observed can be calculated using the survival function as:

chi2_rv1.sf(2.69)

0.2605397078599757

Note the the value from analysis is approximately equal to the value found through resampling.

As usual, there are also methods that can reduce our effort in determining the \(p\)-value. The function stats.chi2_contingency() takes as input a contingency table and returns:

  • the chi-squared statistic,

  • the analytical \(p\)-value,

  • the number of degrees of freedom, and

  • the expected contingency table.

stats.chi2_contingency(contingency)

(2.686121212121211,
 0.26104548728461613,
 2,
 array([[ 8.42553191,  2.34042553, 11.23404255],
        [ 9.57446809,  2.65957447, 12.76595745]]))

Note that the \(p\)-value slightly differs from our previous analysis. The reason is that the statistic is not really a chi-squared random variable under \(H_0\) because we are processing a set of discrete values and representing the result by a continuous distribution. To compensate for this, stats.chi2_contingency() applies an adjustment called Yate’s continuity correction to produce a more accurate estimate of the true \(p\)-value.

To find the \(p\)-value without the continuity correction, pass the correction=False argument:

stats.chi2_contingency(contingency, correction=False)

(2.686121212121211,
 0.26104548728461613,
 2,
 array([[ 8.42553191,  2.34042553, 11.23404255],
        [ 9.57446809,  2.65957447, 12.76595745]]))

Now the result matches our analysis exactly. In practice, it is better to use the continuity correction.

Now let’s use stats.chi2_contingency() to calculate the \(p\)-value for the data features sex and exercised. Recall that the contingency table is

ex_contingency

There is \((2-1)(2-1)=1\) degree of freedom, so we can calculate the probability of seeing such a large chi-squared statistic as:

chi2_rv2 = stats.chi2(1)

0.008360480726183703

We pass this as the argument of stats.chi2_contingency():

stats.chi2_contingency(ex_contingency, correction=False)

(6.954582494119101,
 0.008360480726183703,

 array([[ 82.71428571, 496.28571429],

The value of the chi-squared statistic and the two \(p\)-values generated by analysis match, but they differ from our simulation result. If we allow the continuity correction, then we get

stats.chi2_contingency(ex_contingency)

(6.534102100555383,

This \(p\)-value is very close to our result via resampling. Again, since \(p \approx 0.011\) is less than our signficance threshold of \(\alpha=0.05\), we reject the null hypothesis and conclude that these data features are dependent.

12.2.3. Fisher’s Exact Test#

For small tables, instead of using random permutations of the data, we can simply use every possible permutation of the data. The resulting test is called Fisher’s Exact Test. Students who wish to implement this test may wish to use the itertools library to generate all permutations of the data for one of the features.
